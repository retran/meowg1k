# Integrations Guide

`meowg1k` is built for automation. Its scriptable, non-interactive design makes it a perfect component for Git hooks, CI/CD pipelines, and custom scripts. This guide provides practical examples for integrating `meowg1k` into your development workflow.

---

## 1. Git Hooks

Git hooks are scripts that run automatically at certain points in the Git lifecycle. They are perfect for automating tasks like generating commit messages or running pre-commit checks.

### Automatically Generating Commit Messages

This is one of the most powerful integrations. The `prepare-commit-msg` hook runs after you type `git commit` but before the editor opens. We can use it to have `meowg1k` generate the commit message for us.

**Goal:** Automatically populate your commit message with a message generated by `meow commit`.

**Setup:**

1. Create a file named `.git/hooks/prepare-commit-msg`.
2. Paste the following script into the file.
3. Make the script executable: `chmod +x .git/hooks/prepare-commit-msg`.

**Script (`.git/hooks/prepare-commit-msg`):**

```bash
#!/bin/bash
#
# Auto-generates a commit message using 'meow commit'.
# The script only runs for regular commits and allows you to edit the message.
#

COMMIT_MSG_FILE=$1
COMMIT_SOURCE=$2

# Only run for regular commits (not merge, squash, amend, etc.)
if [ -z "$COMMIT_SOURCE" ]; then
    echo "meowg1k: Generating commit message..."
    # Use 'meow commit' and write the output to the commit message file
    meow commit --silent > "$COMMIT_MSG_FILE"
fi
```

Now, when you run `git commit`, your editor will open pre-filled with a generated message, ready for you to review and save.

### Pre-commit Code Analysis

The `pre-commit` hook runs after you type `git commit` but before the `prepare-commit-msg` hook. It's useful for running quick checks on the files you're about to commit.

**Goal:** Run a quick security check on staged Go files before committing.

**Script (`.git/hooks/pre-commit`):**

```bash
#!/bin/bash

echo "meowg1k: Running pre-commit analysis..."

# Get a list of staged Go files
STAGED_GO_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep '\.go$')

if [ -z "$STAGED_GO_FILES" ]; then
    echo "No Go files to analyze."
    exit 0
fi

for file in $STAGED_GO_FILES; do
    echo "Analyzing $file..."
    # Pipe the file content to meowg1k for a quick security check
    # The output is shown but does not block the commit.
    # To block the commit on issues, you would add logic to exit 1 on failure.
    cat "$file" | meow g -u "Quick security check: any obvious issues in this Go code?"
done

exit 0
```

## 2. CI/CD Pipelines

Integrating `meowg1k` into your CI/CD pipeline ensures consistency and automates tasks for your entire team.

### GitHub Actions: Auto-generate PR Descriptions

This workflow automatically generates a PR description when a new pull request is opened.

**Goal:** When a PR is opened, a GitHub Action runs `meow pr` and updates the PR's body with the generated content.

**Setup:** Create a file at `.github/workflows/pr_description.yml`.

**Workflow (`.github/workflows/pr_description.yml`):**

```yaml
name: Auto PR Description

on:
  pull_request:
    types: [opened]

permissions:
  pull-requests: write
  contents: read

jobs:
  generate-pr-description:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0 # Required to compare branches

      - name: Setup Go
        uses: actions/setup-go@v5
        with:
          go-version: '1.25.1'

      - name: Install meowg1k
        run: go install github.com/retran/meowg1k@latest

      - name: Generate PR Description
        id: generate
        run: |
          DESCRIPTION=$(meow pr --base ${{ github.base_ref }} --silent)
          # Use heredoc to properly handle multiline descriptions
          echo "DESCRIPTION<<EOF" >> $GITHUB_OUTPUT
          echo "$DESCRIPTION" >> $GITHUB_OUTPUT
          echo "EOF" >> $GITHUB_OUTPUT
        env:
          # Store your API key as a secret in your repository settings
          MEOW_GEMINI_API_KEY: ${{ secrets.MEOW_API_KEY }}

      - name: Update PR Description
        run: gh pr edit ${{ github.event.pull_request.number }} --body "${{ steps.generate.outputs.DESCRIPTION }}"
        env:
          # The GITHUB_TOKEN is automatically provided by GitHub Actions
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## 3. Custom Shell Scripts

Because `meowg1k` follows Unix principles (reading from stdin, writing to stdout), it's easy to use in any custom shell script.

### Batch Code Review Script

This script iterates through all Go and Python files in a project, runs a review on each one, and saves the output to a results directory.

**Goal:** Perform a bulk analysis of an entire codebase.

**Script (`run_batch_review.sh`):**

```bash
#!/bin/bash
#
# A script to analyze all Go and Python source files in a directory.
#

set -euo pipefail

TARGET_DIRECTORY=${1:-.} # Default to current directory
OUTPUT_DIR="analysis_results"

mkdir -p "$OUTPUT_DIR"
echo "Starting batch analysis. Results will be saved in: $OUTPUT_DIR"

# Find all .go and .py files, excluding vendor directories
find "$TARGET_DIRECTORY" -type f \( -name "*.go" -o -name "*.py" \) -not -path "*/vendor/*" | while read -r file; do
    echo "Analyzing: $file"

    # Generate a clean output filename
    output_filename=$(echo "$file" | tr '/' '_')

    # Run the analysis and save the output
    cat "$file" | meow g -u "Perform a thorough code review and suggest improvements." --silent > "$OUTPUT_DIR/$output_filename.md"
done

echo "Batch analysis complete!"

```
